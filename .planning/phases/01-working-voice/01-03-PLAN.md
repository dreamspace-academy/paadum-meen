`frontend/src/state/kioskStateMachine.ts`:
- Import `KioskState`, `KioskEvent`, `KioskTransition` from `./kioskTypes`.
- Export `const VALID_TRANSITIONS: KioskTransition[]` — define all valid transitions from api-contract.md §11.2:
  - idle + SPACE_DOWN → listening
  - listening + SPACE_UP → thinking
  - listening + ERROR → error
  - thinking + AUDIO_STARTED → speaking
  - thinking + ERROR → error
  - speaking + RESPONSE_DONE → returning
  - speaking + ERROR → error
  - returning + SPACE_DOWN → listening
  - returning + RESPONSE_DONE → idle (idle timeout path — timer-driven)
  - error + RESET → idle
  - returning + RESET → idle
- Export `function canTransition(from: KioskState, event: KioskEvent): boolean`.
- Export `function transition(from: KioskState, event: KioskEvent): KioskState`:
  - Find matching entry in VALID_TRANSITIONS.
  - If not found: log `[state] blocked transition ${from} + ${event}` and return `from` unchanged.
  - Log `[state] ${from} -> ${to}` on every valid transition.
  - Return `to`.
- Guard: if `event === 'SPACE_DOWN' && from === 'listening'`, log `[keyboard] ignored repeated Space keydown` and return `from`.
- Guard: if `event === 'SPACE_UP' && from !== 'listening'`, log `[keyboard] ignored stale keyup because state=${from}` and return `from`.

`frontend/src/api/backendClient.ts`:
- Read base URL from `import.meta.env.VITE_BACKEND_BASE_URL` (falls back to `http://localhost:8000`).
- Export `async function createRealtimeSession(clientSessionId: string, reason: string): Promise<RealtimeSessionResponse>`:
  - POST to `${BASE_URL}/api/realtime/session` with `Content-Type: application/json`.
  - Body: `{ clientSessionId, reason }`.
  - On non-2xx: throw `Error` with code from response JSON `error.code`.
  - CRITICAL: This function must NOT log the `clientSecret` field. Log only `session.id` on success.
- Export `async function postReset(body: ResetRequest): Promise<ResetResponse>`:
  - POST to `${BASE_URL}/api/session/reset`.
  - If the request fails (network error or non-2xx), log the error but do NOT throw — frontend resets locally regardless (per api-contract.md §7.6).
  - Return the response JSON or a synthetic `{ status: 'error', reset: { accepted: false } }` on failure.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Keyboard events → state machine | window keydown/keyup events from untrusted visitor input drive state transitions |
| OpenAI data channel → App.tsx | realtime events from WebRTC data channel drive state; untrusted event shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | kioskStateMachine.ts | mitigate | transition() is a pure function; invalid from+event combos return current state unchanged; stale keyup guard prevents invalid transition from post-reset keyup |
| T-03-02 | Denial of Service | keyboard handler in App.tsx | mitigate | Repeated Space keydown while in listening state is blocked by repeat-keydown guard and logged — no duplicate getUserMedia calls |
| T-03-03 | Information Disclosure | backendClient.ts | mitigate | createRealtimeSession() logs only session.id, never clientSecret; grep gate in acceptance_criteria |
| T-03-04 | Tampering | OpenAI data channel onmessage | mitigate | Only specific event types (response.audio.delta, response.done, error) trigger state transitions; unrecognized events are logged but ignored |
| T-03-SC | Tampering | npm packages | accept | No new npm packages installed in this plan — only project scaffold packages from Plan 01 |
</threat_model>
