## Phase Goal

**As a** library visitor, **I want to** hold Space to speak with the singing fish and hear it respond by voice, **so that** I can learn about MakerSpace, DreamSpace, and Batticaloa through natural speech conversation.

<objective>
Scaffold the entire project structure (backend Python + frontend React) and implement the thinnest possible end-to-end voice loop: backend creates an OpenAI Realtime ephemeral session; frontend receives the client secret, establishes WebRTC, captures mic on Space keydown, commits on keyup, and plays the AI voice response. This proves the core architecture works.

Purpose: Remove all architectural risk before building out the full kiosk shell. If the WebRTC ephemeral token flow does not work, we know immediately — not after building 300 lines of UI.

Output: Two runnable processes (`uvicorn` at :8000, `vite` at :5173) that together deliver a complete voice round-trip.
</objective>

TYPES (inline, no separate file for skeleton):
```
type KioskState = 'idle' | 'listening' | 'thinking' | 'speaking' | 'returning' | 'error';
```

STATE: `const [kioskState, setKioskState] = useState<KioskState>('idle');`

SESSION INIT (useEffect on mount):
- Generate `clientSessionId` as `pm_${Date.now()}`.
- POST to `${import.meta.env.VITE_BACKEND_BASE_URL}/api/realtime/session` with body `{clientSessionId, reason: 'app_start'}`.
- On success: parse response, store `clientSecret`, `sessionId`, `expiresAt` in refs.
- Create `RTCPeerConnection` with iceServers `[{ urls: 'stun:stun.l.google.com:19302' }]`.
- Fetch OpenAI SDP offer: `POST https://api.openai.com/v1/realtime?model=gpt-4o-realtime-preview-2024-12-17` with `Authorization: Bearer {clientSecret}`, `Content-Type: application/sdp`, body = `pc.localDescription.sdp`.
- Create data channel named `oai-events` BEFORE calling `createOffer`.
- Set up data channel `onmessage` handler for events: `response.audio.delta`, `response.done`, `error`.
- For `response.audio.delta`: decode base64 audio data and schedule playback via `AudioContext`. Set state to `speaking` if not already.
- For `response.done`: log `[realtime:event] response.done`, transition to `returning`.
- For `error`: log `[realtime:event] error`, transition to `error`.
- On `pc.ontrack` event: connect remote audio stream to an `<audio autoPlay>` element in the DOM or play via AudioContext — either approach is acceptable for skeleton.
- After signaling completes and data channel opens (`ondataopen`), call `setKioskState('idle')`.
- On any fetch/WebRTC failure: `setKioskState('error')`.
- NEVER log `clientSecret` to console. Log only `sessionId` on success.

KEYBOARD HANDLERS (useEffect with addEventListener on window):
- `keydown` on `Space`:
  - If `kioskState === 'listening'`: log `[keyboard] ignored repeated Space keydown`, return.
  - If `kioskState !== 'idle' && kioskState !== 'returning'`: return silently.
  - Get `getUserMedia({ audio: true })`. On permission denied: `setKioskState('error')`.
  - Add audio track to `pcRef.current` as a sender.
  - `setKioskState('listening')`.
  - Log `[state] ${prev} -> listening`.
- `keyup` on `Space`:
  - Stale keyup guard (from api-contract.md §11.4): `if (kioskState !== 'listening') { console.debug('[keyboard] ignored stale keyup because state=', kioskState); return; }`.
  - Remove audio senders from RTCPeerConnection (call `pc.removeTrack(sender)` for each sender).
  - Stop all tracks in the stream (`stream.getTracks().forEach(t => t.stop())`).
  - `setKioskState('thinking')`.
  - Log `[state] listening -> thinking`.

RENDER: For the skeleton, render a minimal dark screen with the current state as centered text. No KioskFrame component yet — that's Plan 03.

```tsx
return (
  <div style={{ display: 'flex', alignItems: 'center', justifyContent: 'center',

  </div>
);
```

Commit `kioskState` to refs where needed inside event listeners to avoid stale closure bugs — use `useRef` mirroring the `useState` value for keyboard handler comparisons.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend → Backend | fetch POST to localhost:8000/api/realtime/session — untrusted client request crosses here |
| Backend → OpenAI | httpx POST with real OPENAI_API_KEY — only crossing where the secret is used |
| OpenAI → Frontend | WebRTC data channel events — untrusted model output enters browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | backend/app/api/realtime.py | mitigate | OPENAI_API_KEY read from env only; never serialized into response JSON or logged; grep gate in acceptance_criteria |
| T-01-02 | Information Disclosure | frontend/src/app/App.tsx | mitigate | clientSecret stored only in useRef, never in console.log, never in React state that could be serialized to DevTools; grep gate enforced |
| T-01-03 | Elevation of Privilege | POST /api/realtime/session | accept | localhost-only MVP; no auth required; no sensitive user data at risk |
| T-01-04 | Tampering | keydown/keyup state machine | mitigate | stale keyup guard implemented per api-contract.md §11.4; repeat keydown guard prevents double state entry |
| T-01-05 | Denial of Service | httpx OpenAI call | mitigate | 15-second timeout on OpenAI session creation; error returned as structured JSON, not unhandled exception |
| T-01-SC | Tampering | npm install | mitigate | Only official packages (react, vite, typescript) from npmjs.com — no third-party packages in skeleton; uv for backend installs fastapi/httpx/uvicorn from pypi.org |
</threat_model>
