# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Working Voice.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope boundary:** Phase 1 delivers the minimal kiosk shell only — portrait 9:16
> frame, a centered instruction, simple text-based state cues (idle / listening /
> thinking / speaking / returning / error), and the error message screen. The full
> animated mascot, Batticaloa background art, and all six animation states are
> deferred to Phase 2. The language selector and admin Reset Session UI are deferred
> to Phase 3. Phase 1 must be brand-consistent with what Phase 2 will build.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | shadcn gate: Phase 1 UI has no standard components — full-screen kiosk shell with text labels only; a component library adds no value here |
| Preset | not applicable | — |

**shadcn gate decision:** The kiosk has no clickable visitor controls and no standard
form/card/modal patterns in Phase 1. The entire UI is a full-screen canvas with
centered text. shadcn would add zero value and unnecessary build complexity. Record:
`Tool: none — developer confirmed kiosk-only context, no shadcn required.`

## Typography

All text must be legible to a child standing 60–90 cm from a monitor. Minimum
effective size for body copy at kiosk distance is 24px. Sizing is expressed in `rem`
with a 16px root; the pixel equivalents are listed for clarity.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Instruction | 36px (2.25rem) | 700 (Bold) | 1.2 | "Hold Space to talk" — primary visitor instruction, center screen |
| State label | 20px (1.25rem) | 400 (Regular) | 1.4 | Current state displayed as text cue (e.g. "Listening…", "Thinking…") |
| Error heading | 28px (1.75rem) | 700 (Bold) | 1.2 | "Something went wrong." — error state first line |
| Error body | 20px (1.25rem) | 400 (Regular) | 1.5 | "Please try again." — error state second line |

Font weights in use: **400** (Regular) and **700** (Bold). Maximum two weights.

Font family declaration:

```css
font-family: 'Inter', system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI',
             sans-serif;
```

Character set requirement: Inter must be loaded with `latin` and `latin-ext` subsets
to support future Tamil and Sinhala transliteration fallback. Phase 3 will add a
proper Tamil/Sinhala web font.

## Color

Color palette is derived from the mascot asset files in `tmp/assets/mascot-parts/`
and the overall brand identity of Batticaloa / DreamSpace.

**Background philosophy:** Phase 1 uses a dark near-black screen (simulating a
nighttime river / kiosk environment) as the dominant surface. This is brand-aligned
with the moonlit Batticaloa river scene that Phase 2 will build. It ensures the text
instruction is highly legible and the kiosk reads as a display, not a website.

| Role | Value | Usage | Source |
|------|-------|-------|--------|
| Dominant (60%) | `#0D1117` | Full-screen background — nighttime river dark | Default; brand-aligned with moonlit scene (Phase 2 replaces with illustrated BG) |
| Secondary (30%) | `#1A2440` | State label container / subtle panel — deep navy | Derived from mascot tail deep royal blue (`~#1A2D8A`), darkened for surface use |
| Accent (10%) | `#E8952A` | Instruction text, active state indicator ring, error heading text | Derived from mascot top garment golden/amber; high contrast on dark background |
| Destructive | `#CC3333` | Not used in Phase 1 visitor UI. Reserved for Phase 3 admin Reset button confirmation state | Derived from mascot garment red accent |
| Text primary | `#F0ECE4` | All body/instruction text on dark surfaces | Warm white — avoids harsh pure white on dark kiosk screen |
| Text muted | `#8A9BAD` | State label text in non-error states | Muted blue-grey — readable but subordinate to instruction |
| Error surface | `#1F0D0D` | Background tint applied behind error message | Dark red-black — communicates error without harsh colors |
| Error accent | `#FF6B6B` | Error state indicator ring / pulse | Legible on dark error surface |

**Accent reserved for:**
1. The "Hold Space to talk" instruction text
2. The active-state indicator element (Phase 1: a simple text label or thin ring; Phase 2: mascot animation cue)
3. The "Something went wrong." error heading text

**Accent is NOT used for:** general body text, state labels, backgrounds, borders, or any decorative element.

**60/30/10 allocation:**
- 60% `#0D1117`: the full viewport background
- 30% `#1A2440`: the centered content panel / state label area
- 10% `#E8952A`: instruction text, active ring, error heading

## State Visual Map

Phase 1 represents fish states with text labels + a subtle visual indicator (a thin
colored ring or pulsing dot above/below the instruction). No mascot animation in
Phase 1.

| State | Label Text | Indicator Color | Instruction Visibility |
|-------|-----------|-----------------|----------------------|
| idle | *(none shown — instruction is the only element)* | none | Visible — full opacity |
| listening | "Listening…" | `#4ADE80` (green pulse) | Dimmed to 40% opacity |
| thinking | "Thinking…" | `#60A5FA` (blue pulse) | Hidden |
| speaking | "Speaking…" | `#A78BFA` (purple pulse) | Hidden |
| returning | "Finishing…" | `#60A5FA` (blue, fading) | Hidden |
| error | *(replaced by error message block)* | `#FF6B6B` (red) | Hidden |

State indicator colors are temporary Phase 1 scaffolding. Phase 2 replaces these
with mascot animation states. These colors are not part of the permanent brand palette.

## Interaction Contract

### Input model

There are **no clickable visitor controls** in Phase 1. The only visitor input is
the Space key.

```
Space keydown  → transition to listening (from idle or returning only)
Space keyup    → transition to thinking (from listening only)
```

### Keyboard guard rules (from REQUIREMENTS.md / system-architecture.md)

1. Ignore repeated `keydown` events (key repeat) if already in `listening` state.
2. Ignore `keyup` events if current state is not `listening` (stale keyup guard).
3. Do not intercept or block any other keys — Space is the only monitored key.

```ts
// Stale keyup guard — source: api-contract.md §11.4
if (event.type === "keyup" && event.code === "Space") {
  if (state !== "listening") return;
  stopListeningAndCommitAudio();
}
```

### Accessibility

This is a public kiosk. Standard WCAG keyboard focus styles are suppressed (no
visitor-facing focus rings needed — the only input is Space). The instruction text
must have contrast ratio ≥ 4.5:1 against `#0D1117`.

Contrast check:
- `#E8952A` on `#0D1117` → luminance contrast ≈ 7.2:1 — **passes WCAG AA and AAA**
- `#F0ECE4` on `#0D1117` → luminance contrast ≈ 14.8:1 — **passes WCAG AAA**
- `#8A9BAD` on `#1A2440` → luminance contrast ≈ 4.6:1 — **passes WCAG AA**

### No-input states

During `thinking`, `speaking`, and `returning`, the Space key has no effect
(state machine guard). The UI provides no visual affordance that would suggest
the visitor should do anything while the fish responds.

## Copywriting Contract

All copy is in English for Phase 1. Phase 3 adds Tamil and Sinhala translations.

| Element | Copy | Notes |
|---------|------|-------|
| Primary instruction (idle) | "Hold Space to talk" | No punctuation — reads as a direct action command |
| State: listening | "Listening…" | Ellipsis communicates ongoing activity |
| State: thinking | "Thinking…" | Ellipsis communicates ongoing activity |
| State: speaking | "Speaking…" | Not shown to visitor in Phase 2 (mascot speaks); in Phase 1 this is a placeholder |
| State: returning | "Finishing…" | Communicates the fish is wrapping up |
| Error heading | "Something went wrong." | Short, non-technical, full stop — source: KIOSK-06 / system-architecture.md §11.1 |
| Error body | "Please try again." | Directs visitor to retry; avoids technical detail |
| Empty state | *(not applicable — idle state always shows the instruction; there is no "no data" condition)* | Voice-only kiosk has no empty data state |
| Destructive confirmation | *(not applicable in Phase 1 — admin Reset button is deferred to Phase 3)* | KIOSK-07 is a Phase 3 requirement |

**Tone:** Short, direct, child-friendly. No jargon. No ellipsis on error messages
(error is a completed state, not ongoing). Ellipsis only on active ongoing states.
