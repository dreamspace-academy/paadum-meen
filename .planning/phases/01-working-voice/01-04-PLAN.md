SESSION INIT (useEffect on mount):
```
async function initSession(reason: string): Promise<void> {
  try {
    const sessionResp = await createRealtimeSession(clientSessionIdRef.current, reason);
    sessionDataRef.current = sessionResp.session;
    const client = new OpenAIRealtimeClient({
      sessionData: sessionResp.session,
      onEvent: handleRealtimeEvent,
      onConnectionStateChange: handleConnectionStateChange,
    });
    clientRef.current = client;
    await client.connect();
    applyEvent('RESET'); // → idle (from initial disconnected state not tracked in machine)
    // setKioskState('idle') directly to ensure we land in idle
  } catch {
    applyEvent('ERROR');
  }
}
initSession('app_start');
```

REALTIME EVENT HANDLER (`handleRealtimeEvent`):
- Parse `event.data` as JSON.
- `response.audio.delta` → `applyEvent('AUDIO_STARTED')`.
- `response.audio.done` → log `[realtime:event] response.audio.done`.
- `response.audio_transcript.delta` → log subtitle text (just log for Phase 1 — no SubtitlePanel yet).
- `response.done` → `applyEvent('RESPONSE_DONE')`; start idle timer.
- `error` → log detailed error; `applyEvent('ERROR')`.
- Unknown event types → log and ignore.

CONNECTION STATE CHANGE HANDLER (`handleConnectionStateChange`):
- If `state === 'failed'` and `stateRef.current !== 'idle' && stateRef.current !== 'error'`:
  - Log `[error] WebRTC connection failed during ${stateRef.current}`.
  - `applyEvent('ERROR')`.
- If `state === 'connected'`: log `[session] WebRTC connected`.

IDLE TIMER:
- `function startIdleTimer(): void`:
  - Clear any existing timer.
  - `idleTimerRef.current = setTimeout(async () => { await onIdleTimeout(); }, IDLE_TIMEOUT_MS)`.
- `function cancelIdleTimer(): void`:
  - If `idleTimerRef.current` is set: `clearTimeout(idleTimerRef.current); idleTimerRef.current = null`.
- `async function onIdleTimeout(): Promise<void>`:
  - Log `[reset] reason=idle_timeout previousState=${stateRef.current}`.
  - Call `postReset({ clientSessionId: clientSessionIdRef.current, reason: 'idle_timeout', previousState: stateRef.current })` — do not await result (fire and forget; postReset already swallows errors internally).
  - `setKioskState('idle')` directly (bypassing state machine — idle timeout is authoritative).
  - Per SAFETY-03: clear any subtitle state here when SubtitlePanel is added in future phases.

KEYBOARD HANDLER (useEffect with addEventListener on window):
- Listen for `keydown` and `keyup` with `{ code: 'Space' }` check.
- Space keydown:
  1. Guard: `if (event.repeat) return;` (browser key-repeat)
  2. Guard: `if (stateRef.current === 'listening') { log(KEYBOARD, 'ignored repeated Space keydown'); return; }`
  3. Guard: `if (stateRef.current !== 'idle' && stateRef.current !== 'returning') return;`
  4. If `stateRef.current === 'returning'`: `cancelIdleTimer()`.
  5. Reconnect check: `if (!clientRef.current?.isConnected() || clientRef.current?.isExpiredSoon())`:
     - `await initSession('connection_recovery')` — if this fails, applyEvent('ERROR') and return.
  6. Acquire mic: `micHandleRef.current = await acquireMicrophone(clientRef.current!)`.
     - On error: `applyEvent('ERROR'); return`.
  7. `applyEvent('SPACE_DOWN')`.
- Space keyup:
  1. Stale keyup guard (from api-contract.md §11.4): `if (stateRef.current !== 'listening') { log(KEYBOARD, 'ignored stale keyup because state=', stateRef.current); return; }`.
  2. `stopMicrophone(micHandleRef.current!, clientRef.current!)`.
  3. `micHandleRef.current = null`.
  4. `applyEvent('SPACE_UP')`.

RESPONSE_DONE handler (inside handleRealtimeEvent for `response.done`):
- `applyEvent('RESPONSE_DONE')` transitions to `returning`.
- Call `startIdleTimer()`.

Cleanup on unmount: close RTCPeerConnection, cancel idle timer, stop mic.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WebRTC data channel → handleRealtimeEvent | OpenAI realtime events are untrusted strings from external model |
| getUserMedia → microphone.ts | Browser-mediated microphone access — must be short-lived |
| Idle timer → postReset | Fire-and-forget backend call on timeout; must not block local reset |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | openaiRealtimeClient.ts connect() | mitigate | clientSecret used as Authorization header value only; log only sessionId; sanitizer in frontendLogger.ts catches accidental leaks |
| T-04-02 | Denial of Service | idle timer | mitigate | clearTimeout() called on every new timer start; only one timer can be active; Space-in-returning cancels timer before re-entering listening |
| T-04-03 | Tampering | stale keyup after reset | mitigate | stateRef.current check in keyup handler; if state !== listening, event is silently dropped and logged |
| T-04-04 | Privacy | microphone capture | mitigate | acquireMicrophone() called only on SPACE_DOWN; stopMicrophone() called on SPACE_UP and unmount; no always-listening; SAFETY-02 |
| T-04-05 | Availability | WebRTC reconnect on connection_recovery | mitigate | initSession('connection_recovery') called before entering listening if !isConnected() or isExpiredSoon(); on initSession failure → error state, not silent hang |
| T-04-06 | Privacy | SAFETY-03 context clear | mitigate | onIdleTimeout() marks the context clear point; SubtitlePanel clear will be called here when added in Phase 2 |
| T-04-SC | Tampering | no new npm packages | accept | This plan adds no new npm dependencies |
</threat_model>
